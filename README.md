# GE-CSG: Group-Equivariant Context-Sensitive Grammars for DNA

A framework for parsing and probabilistic modelling of coding DNA sequences using **group-equivariant formal grammars** (GE-CSG). The grammar structure mirrors the algebraic symmetry group acting on codons, yielding compact rule representations and biologically meaningful parse trees.

---

## Background

### The Symmetry Group of Codons

DNA codons (triplets over {A, T, G, C}) admit a natural symmetry group

$$G = \mathbb{Z}_2^{\text{comp}} \times \mathbb{Z}_2^{\text{rev}} \times \mathbb{Z}_3$$

of order 12, generated by:

| Generator | Action |
|---|---|
| $\mathbb{Z}_2^{\text{comp}}$ | Watson-Crick complement (A↔T, G↔C) |
| $\mathbb{Z}_2^{\text{rev}}$ | Sequence reversal |
| $\mathbb{Z}_3$ | Cyclic frame shift |

The stabiliser subgroup $G_i < G$ of a single codon position has index 4, yielding **four cosets** (coset space $G/G_i$):

| Coset | First nucleotide | Biological role |
|---|---|---|
| C0 | A__ | Start codons (ATG), A-body |
| C1 | T__ | Stop codons (TAA, TAG, TGA), T-body |
| C2 | G__ | G-body codons |
| C3 | C__ | C-body codons |

### G-Equivariance

A grammar is **G-equivariant** if every production rule is closed under the group action. In practice, a single *orbit representative* rule automatically generates all equivalent rules for every coset. This gives:

- **Compression**: 7 orbit-representative rules expand to 16 full rules + 64 breaking rules
- **Biological consistency**: the grammar respects the codon symmetry structure by construction

---

## Complete DNA Grammar

The grammar encodes a full biological coding sequence (CDS):

```
Gene       →  CDS
CDS        →  StartCodon  ORF  StopCodon
ORF        →  BodyCodon
           |  BodyCodon  ORF
StartCodon →₁ C0          (orbit-expands to all 4 cosets)
StopCodon  →₁ C1
BodyCodon  →₁ C0
```

Each `→₁` rule is an **orbit representative**: the GE-CSG framework lifts it to rules for all cosets via the group action. A minimal valid CDS is 3 codons: `ATG + ≥1 body codon + TAA/TAG/TGA`.

### Parsing: G-Equivariant Earley

The parser (`EquivariantEarleyParser`) runs standard Earley chart parsing over the lifted grammar. For a sequence of length $n$ codons it operates on the coset sequence rather than raw nucleotides, giving a parse tree annotated with coset labels at every leaf.

## Stochastic Extension

The stochastic grammar adds weights to production rules, turning parse trees into probability distributions over CDS sequences.

### Probability Model

| Rule | Weight |
|---|---|
| `ORF → BodyCodon` | $p_{\text{terminal}}$ |
| `ORF → BodyCodon ORF` | $1 - p_{\text{terminal}}$ |
| `NT →₁ Cᵢ` (coset terminal) | $0.25$ (orbit size = 4) |
| Breaking rule for codon $w$ in coset $C_k$ | $p(w)$ |

The breaking rule probability $p(w)$ can be:
- **Uniform**: $1/16$ for all 64 codons
- **Human codon usage**: empirical frequencies (Kazusa database), normalised per coset to sum to 1

### Log-Probability (Viterbi)

Because the grammar is unambiguous (each accepted sequence has exactly one parse tree), the Viterbi log-probability equals the full parse probability:

$$\log P(\text{seq}) = \sum_{\text{rules}} \log w(r) + \sum_{\text{codons}} \log p(w)$$

The `StochasticEarleyParser` computes this by a single tree walk, and exposes per-node log-probability contributions for interpretability.

---

## Performance

Empirical complexity measured by sweeping CDS lengths 1–50 body codons (10 reps each):

| Parser | Mean (ms) | Complexity (log-log slope) |
|---|---|---|
| `earley_deterministic` | ~5–6 | ≈ O(n²) |
| `earley_stochastic_uniform` | ~4 | ≈ O(n²) |
| `earley_stochastic_human` | ~4 | ≈ O(n²) |

## Quick Start

```bash
conda activate genom

# Run all tests
python -m unittest discover -s test -p "test_*.py"

# Generate parse tree images (complete grammar)
python visualization/visualize_all.py

# Generate stochastic parse trees
python visualization/visualize_stochastic.py

# Collect performance data
python run_profiler.py --max 50 --reps 10

# Visualise performance
python visualization/visualize_profile.py
```
